# 05_UI_UX_Guidelines.md

# UI / UX Guidelines

## Purpose

Define a consistent design system and user experience for the Health &
Fitness Coach application.

------------------------------------------------------------------------

# Design Principles

-   Simple
-   Modern
-   Minimal
-   Fast
-   Accessible
-   Consistent

------------------------------------------------------------------------

# Theme

Support:

-   Light Mode
-   Dark Mode

System theme should be supported.

------------------------------------------------------------------------

# Color Palette

Primary: - Green

Secondary: - Blue

Success: - Green

Warning: - Orange

Error: - Red

Background: - White / Dark Gray

------------------------------------------------------------------------

# Typography

Headings: - Bold - Large

Body: - Medium weight

Caption: - Small

Use a single font family throughout the app.

------------------------------------------------------------------------

# Spacing

Use an 8pt spacing system.

-   4
-   8
-   16
-   24
-   32
-   48

------------------------------------------------------------------------

# Border Radius

-   Cards: 16
-   Buttons: 12
-   Inputs: 12
-   Dialogs: 20

------------------------------------------------------------------------

# Icons

Use one icon library consistently.

Recommended: - Cupertino - Material Symbols

------------------------------------------------------------------------

# Buttons

Types

-   Primary
-   Secondary
-   Outlined
-   Text
-   Icon
-   Floating Action Button

States

-   Normal
-   Loading
-   Disabled

------------------------------------------------------------------------

# Input Fields

Support

-   Label
-   Hint
-   Validation
-   Prefix Icon
-   Suffix Icon

------------------------------------------------------------------------

# Cards

Reusable cards:

-   Workout Card
-   Nutrition Card
-   Statistic Card
-   Progress Card
-   Meal Card

------------------------------------------------------------------------

# Navigation

Bottom Navigation

-   Home
-   Workout
-   Nutrition
-   Progress
-   Profile

------------------------------------------------------------------------

# Dashboard

Widgets

-   Calories
-   Water
-   Weight
-   BMI
-   Streak
-   Workout Progress

------------------------------------------------------------------------

# Animations

Use subtle animations.

Examples

-   Fade
-   Slide
-   Scale
-   Hero
-   Progress

Avoid excessive motion.

------------------------------------------------------------------------

# Loading States

Use

-   Skeleton Loader
-   Circular Progress
-   Shimmer

------------------------------------------------------------------------

# Empty States

Provide

-   Illustration
-   Helpful Message
-   Action Button

------------------------------------------------------------------------

# Error States

Show

-   Friendly message
-   Retry button

Avoid technical messages.

------------------------------------------------------------------------

# Forms

Validate

-   Email
-   Password
-   Required Fields
-   Numeric Input

------------------------------------------------------------------------

# Accessibility

-   Large touch targets
-   High contrast
-   Screen reader labels
-   Dynamic text scaling

------------------------------------------------------------------------

# Responsive Design

Support

-   Mobile
-   Tablet
-   Web

Use adaptive layouts.

------------------------------------------------------------------------

# Reusable Components

-   AppBar
-   Bottom Navigation
-   Buttons
-   TextField
-   Dialog
-   Snackbar
-   Avatar
-   Badge
-   Charts
-   Progress Ring

------------------------------------------------------------------------

# UX Rules

-   Minimize taps.
-   Keep flows under 5 steps.
-   Confirm destructive actions.
-   Save progress automatically.
-   Give instant feedback.

------------------------------------------------------------------------

# Definition of Done

-   Design system implemented.
-   Components reusable.
-   Responsive layouts.
-   Accessibility verified.
-   Dark mode supported.
-   Consistent spacing and typography.
